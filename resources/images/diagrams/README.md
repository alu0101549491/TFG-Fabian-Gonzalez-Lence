# Diagram Images

This directory contains the rendered PNG exports of the UML class diagrams and use case diagrams generated for the five projects developed as part of the TFG *"Análisis de Capacidades de la IA en la Generación Semiautomática de Aplicaciones Complejas"* (ULL, 2025–2026).

## Purpose

These images are the visual output of the AI-assisted design phase. Each diagram was generated from the corresponding requirement specification selected as definitive and then exported as a PNG for inclusion in the written report and presentation. They serve as a concrete, human-readable record of the design decisions produced by the AI at each project's design stage.

## Structure

Each file follows the naming convention `<project>-<diagram-type>-diagram.png`:

| File | Project | Diagram type |
|---|---|---|
| `hangman-class-diagram.png` | 1 — The Hangman Game | UML Class Diagram |
| `hangman-use-case-diagram.png` | 1 — The Hangman Game | UML Use Case Diagram |
| `player-class-diagram.png` | 2 — Music Web Player | UML Class Diagram |
| `player-use-case-diagram.png` | 2 — Music Web Player | UML Use Case Diagram |
| `balatro-class-diagram.png` | 3 — Mini Balatro | UML Class Diagram |
| `balatro-use-case-diagram.png` | 3 — Mini Balatro | UML Use Case Diagram |
| `carto-class-diagram.png` | 4 — Cartographic Project Manager | UML Class Diagram |
| `carto-use-case-diagram.png` | 4 — Cartographic Project Manager | UML Use Case Diagram |
| `tennis-class-diagram.png` | 5 — Tennis Tournament Manager | UML Class Diagram |
| `tennis-use-case-diagram.png` | 5 — Tennis Tournament Manager | UML Use Case Diagram |

Each diagram shown here corresponds to the final version selected after comparing the diagrams generated from the different formalization levels of the requirements (informal, semi-formal, EARS / user stories). The full multi-version generation logs are stored in [`../../uml-class-diagrams/`](../../uml-class-diagrams/) and [`../../uml-use-case-diagrams/`](../../uml-use-case-diagrams/).

## Usage in the TFG

These images are embedded directly in the written report (chapters 6, *Arquitectura y Diseño*) and in the defence presentation to illustrate the structural and behavioural design of each project as produced by the AI.
