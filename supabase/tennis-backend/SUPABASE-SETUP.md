# Tennis Supabase Setup Guide

> **Following CARTO's approach**: Using Supabase PostgreSQL instead of Render database

---

## Step 1: Create Supabase Project (Free Tier)

1. Go to: **https://supabase.com/dashboard**
2. Click **"New Project"**
3. Fill in details:
   ```
   Organization: TFG (or your organization)
   Name: tennis-tournament-manager
   Database Password: [Generate a strong password - SAVE IT!]
   Region: Europe West (Frankfurt) - Same as Render
   Plan: Free
   ```
4. Click **"Create new project"** (takes ~2 minutes)

---

## Step 2: Get Database Connection String

Once project is created:

1. Go to: **Project Settings** → **Database**
2. Scroll to **"Connection string"**
3. Select: **"Session pooler"** (Port 5432 - NOT Transaction pooler!)
4. Mode: **Session**
5. Copy the connection string:
   ```
   postgresql://postgres.xxx:[YOUR-PASSWORD]@aws-0-eu-central-1.pooler.supabase.com:5432/postgres
   ```
6. **Replace `[YOUR-PASSWORD]`** with the database password you created
7. **SAVE THIS** - you'll need it for Render

## Step 3: Generate SQL Schema from TypeORM

Tennis uses TypeORM entities. We need to generate SQL from them.

### Option A: Use TypeORM Synchronize (Simplest)

```bash
cd /home/fabian/MyStuff/TFG-Fabian-Gonzalez-Lence/projects/5-TennisTournamentManager/backend

# Create temporary .env for local database
cat > .env.temp << 'EOF'
NODE_ENV=development
DATABASE_URL=postgresql://postgres:[PASSWORD]@aws-0-eu-central-1.pooler.supabase.com:5432/postgres
DB_SYNC RONIZE=true
LOG_LEVEL=debug
EOF

# Run migrations to create schema
npx tsx src/infrastructure/database/migrate.ts
```

This will create all tables in Supabase automatically!

## Step 5: Configure Render Backend

Now go back to Render:

1. **Dashboard** → **New** → **Web Service**
2. **OR** if using Blueprint: Let it create `tennis-backend` automatically

3. Set environment variables in Render:

```bash
# Core Config
NODE_ENV=production
PORT=10000
API_PREFIX=/api

# DATABASE (From Supabase Step 2)
DATABASE_URL=postgresql://postgres.xxx:[YOUR-PASSWORD]@aws-0-eu-central-1.pooler.supabase.com:5432/postgres
DB_SYNCHRONIZE=false  # IMPORTANT: false in production!
DB_SSL=true

# CORS (Your GitHub Pages)
CORS_ORIGIN=https://alu0101549491.github.io
SOCKET_CORS_ORIGIN=https://alu0101549491.github.io
APP_URL=https://alu0101549491.github.io

# JWT (Auto-generated by Render)
JWT_SECRET=[auto-generated]
JWT_REFRESH_SECRET=[auto-generated]
```

4. Click **"Save Changes"** → This triggers deployment

## Troubleshooting

### "Database connection failed"

**Check:**
- DATABASE_URL correct (Session pooler, port 5432)?
- Password has special chars? → URL encode them
- DB_SSL=true set?

**Fix:** Verify connection string in Supabase → Database → Connection Strings

### "ECONNREFUSED"

**Check:**
- Using Session pooler (port 5432) NOT Transaction pooler (port 6543)?

### "Relation does not exist"

**Check:**
- Did migrations run? Check Supabase Table Editor
- Is DB_SYNCHRONIZE=false? (should be false in production)

**Fix:** Run migrations manually:
```bash
# In Render Shell or locally
npm run db:migrate
```
